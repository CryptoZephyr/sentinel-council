SENTINEL COUNCIL — PROJECT DESCRIPTION
Bitget AI Base Camp Hackathon S1 — Track 1: Trading Agent
Submission-ready. Under 200 words.
═══════════════════════════════════════════════════════════════════════

Sentinel Council is an autonomous multi-analyst trading agent that
synthesizes five independent market perspectives before simulating any
trade, built on Bitget market data infrastructure.

Problem: Single-source trading agents fail when market regimes shift.
Sentinel Council fixes this with a five-analyst council — the same
approach professional trading desks use, fully automated.

Strategy loop:

PERCEPTION — Every hour, five analysts run against live Bitget futures
data: Macro (BTC ticker + funding rate), Technical (EMA9/21 + RSI from
50 live candles), Sentiment (Fear & Greed contrarian + funding
pressure), News (keyword balance across 4 crypto RSS feeds), and
Market Intel (24h price delta + open interest). All via Bitget MCP.

DECISION — A Council Engine weights the five scores (Macro 30%,
Technical 30%, Sentiment 20%, News 10%, Intel 10%) into a single
confidence figure. >=58 = BUY, <=42 = SELL, else HOLD.

RISK — Position size: 1% of balance (confidence 58-84) or 2% (85+).
Max one position per asset, six concurrent maximum.

EXECUTION — Simulated positions execute against real Bitget price
feeds. Every decision logged to a transparent audit trail.

Bitget modules used: Bitget MCP Server (futures_get_ticker,
futures_get_candles, futures_get_funding_rate,
futures_get_open_interest), Bitget REST API

Word count: ~185 words.
